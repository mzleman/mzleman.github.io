浮动

浮动的作用

浮动主要作用是使元素可以按顺序水平排列，制作一些水平方向的布局效果。

浮动可选值

float: none; 默认

float: left;

float: right;

浮动的特点

  1. 浮动的元素会向父元素内容区的左边界或右边界对齐，多个连续的浮动元素会依次排列。

    float:left;时，从左至右，从上至下依次排列。

    float:right;时，从右至左，从上至下依次排列。

    默认情况下，浮动元素在水平方向不会从父元素中溢出

  2. 元素浮动后，元素性质会发生改变。

    不论是内联元素、块元素还是其他元素，表现一致。

    • 内容区宽度和高度默认由内容撑开。
    • 设置width和height有效。
    • 设置margin、padding、border都有布局效果。
    • 文档流水平方向等式不再要求成立。

  3. 浮动元素与前一个兄弟元素的关系。

    • 如果前一个元素也发生浮动，不论前一个元素的浮动方向是否与自己相同，都会尝试与前一个浮动元素同在一行（只要父元素一行内的剩余宽度足够）。如果该行剩余宽度不足够，则换行浮动。

      剩余宽度足够的标准是：

      剩余宽度 >= 该元素的margin + border + padding + content ⭐ 要考虑左右margin

    • 如果前一个元素没有发生浮动，不论前一个元素是内联元素、块元素还是行内块元素，则会在前一个元素的下一行进行浮动。

      ❗ 注意点：

      (1) 如果前一个未浮动元素是内联元素且包裹文字，则浮动元素会在第一行文字的下方开始浮动。且，如果前一个内联元素中的文字发生了换行，后一个浮动元素默认不会覆盖前一个元素的文字。

      (2) 如果前一个未浮动元素设置了margin-bottom，则后一个浮动元素要考虑前一个元素的该外边距，再换行浮动。

  4. 浮动元素与后一个兄弟元素的关系。

    主要是指如果前一个元素浮动而后一个元素不浮动，则后一个元素上移，因为浮动元素脱离了文档流。

margin collapsing

现象概述

margin collapsing翻译后可以叫“外边距折叠”或“外边距坍塌”。

CSS布局中有一个现象，就是两个或多个块元素之间或块元素与内联元素之间的上外边距(margin-top)和下外边距(margin-bottom)有时会合并（折叠）为单个边距，其大小视情况而定，这种现象称之为“外边距折叠”。

经过在Chrome中测试，块元素与行内块元素之间不发生外边距重叠。❓

四种基本折叠情景

基本前提：只考虑块元素、内联元素，不考虑行内块元素。因为在Chrome中测试，行内块元素与其他兄弟元素之间不发生外边距折叠。

1. 相邻兄弟元素之间的外边距折叠

前一个元素的margin-bottom和后一个元素的margin-bottom会发生折叠现象。

  • 如果二者的相邻外边距都为正，则折叠后的外边距取较大值。
  • 如果二者的相邻外边距一正一负数，则折叠后的外边距取两者之和。
  • 如果二者的相邻外边距都是负数，则折叠后的外边距取两者的较小值（绝对值更大的负数）。

相邻的兄弟元素必然处于同一BFC中，一般情况都会发生外边距的重叠。除非相邻兄弟元素中有发生浮动或者绝对定位。

2. 父元素和其第一个相邻的文档流中的子元素之间的外边距穿透

如果一个父元素和其某个子元素同时满足以下条件：

  1. 父级元素没有border-top、padding-top，或者其计算宽度为0；

  2. 在父元素文档流中的该子元素前，没有哥哥节点在垂直方向上占据空间；

    • 情况1：前面没有任何哥哥元素节点和文本节点。
    • 情况2：前面有哥哥元素节点，但它们都不具有垂直向的尺寸。
    • 情况3：前面有哥哥元素节点，但它们脱离了文档流（浮动、绝对定位）。

  3. 在父元素最上方与该子元素之间，没有不占空间的BFC块级格式化上下文。⭐

    简单理解为，BFC不会被穿透。

  4. 该子元素前面没有元素设置了clear样式而确实隐式增加了一个非零的margin-top。

  5. 该子元素设置了margin-top且非零。

则子元素的上外边距会穿透父容器。

  1. 父级元素没有border-bottom、padding-bottom，或者其计算宽度为0；

  2. 父元素没有设置height和min-height，即父元素的高度默认由内容撑开。

BFC

概念

BFC（Block Formatting Context）格式化上下文,指一个独立的渲染区域或者说是一个隔离的独立容器。

可以把 BFC 理解为一个封闭的大箱子，箱子内部的元素无论如何翻江倒海，都不会影响到外部，同时外部元素也不会干扰内部的布局关系。

开启BFC的元素和方式

  1. 根元素标签默认创建一个BFC。⭐⭐

  2. 浮动的元素会创建一个BFC。⭐⭐

  3. 绝对定位和固定定位的元素创建一个BFC，postion属性为absolute或fixed。⭐⭐

  4. 行内块元素（或元素的 display为 inline-block）。⭐⭐

  5. 表格单元格标签（或元素的display为table-cell）。⭐⭐

  6. 其他表格元素隐式创建的表格单元格，如

    标签、标签、标签、标签、标签都会隐式地创建表格单元格（或元素的display属性为table、table-row、table-row-group、table-header-group、table-footer-group）

  7. overflow样式值不为visible和clip的元素。⭐⭐

  8. 弹性元素，即弹性盒子的直接子元素（display属性值为flex或inline-flex的元素的直接子元素）。

  9. 网格元素，即网格盒子的直接子元素（display属性值为grid或inline-grid的元素的直接子元素）。

  10. …..

    11. 注意：

    1. display: flex/grid; 的元素本身创建的是flex formatting context和grid formatting context，而flex items和grid items会默认创建一个BFC，除非flex items和grid items自己主动创建其他类型的上下文。

    2. 根据MDN的解释，display: table;的元素本身没有创建一个BFC，而是其内部默认会有一个tbody，tbody创建了属于自己的BFC。

定位 position

可选值

  1. static 默认值
  2. fixed 固定定位
  3. relative 相对定位
  4. absolute 绝对定位
  5. sticky 粘滞定位

固定定位 fixed

元素开启固定定位的特点：

  1. 元素脱离文档流，不论是何种元素，元素性质发生以下改变：

    • 宽度、高度默认由内容撑开。
    • 设置width、height有效。
    • 上下左右margin、padding都起到布局作用。

  2. 元素的定位原点在浏览器布局视口的左上角。

    需要好好理解的一句话：如果一个元素开启固定定位，则首先对其他元素进行布局。但如果该定位元素不设置left/right/top/bottom，则该元素会布局在它原本在文档流中的位置（相对于未开启定位前的包含块），并提高层级后进行显示。

  3. 元素的显示层级会提升一层。

  4. 最大的特点：由于定位原点是视口的左上角，所以不论页面如何滚动，该元素的位置都不会发生改变。

相对定位 relative

元素开启相对定位的特点：

  1. 元素不会脱离文档流，元素的性质不会发生改变（块还是块、内联还是内联）。

  2. 元素定位的原点是其原来位置（文档流中/浮动后的位置）。

    开启相对定位的元素不论怎么移动，其盒模型所占的空间（包括margin）仍然会被保留。

    浮动后的元素也可以开启相对定位，设置它的left（假如left > 0），则它会带动其右侧的浮动元素，一起向右移动。设置它的right（假如right > 0），则它会带动其左侧的浮动元素，一起向左移动。

  3. 元素的显示层级会提升一层。

绝对定位 absolute⭐

元素开启绝对定位的特点：

  1. 元素脱离文档流，不论是何种元素，元素性质发生以下改变：

    • 宽度、高度默认由内容撑开。
    • 设置width、height有效。
    • 上下左右margin、padding都起到布局作用。

  2. 元素的定位原点在其包含块的左上角。对于开启绝对定位的元素，其包含块是最近一个position属性不为static的祖先元素，如果没有任何祖先元素开启定位，则其包含块为初始包含块，即浏览器布局视口。

    记忆点1：对于没有开启绝对定位、固定定位的元素，其包含块可以理解为各个元素offsetParent属性指向的节点。

    • 一般情况下，包含块（offsetParent)是离自身最近的祖先块元素。
    • 对于表格内部tbody、tr、td、th等元素，包含块（offsetParent)是其外层的table元素。
    • body、html元素的包含块（offsetParent）是空。

    记忆点2：如果一个元素开启绝对定位，则首先对其他元素进行布局。但如果该定位元素不设置left/right/top/bottom，则该元素会布局在它原本在文档流中的位置（相对于未开启定位前的包含块），并提高层级后进行显示。👈 类似于固定定位。

  3. 元素的显示层级会提升一层。

  4. 元素产生新的水平方向的布局等式，以及垂直布局等式。

    (1) 如果发生过度约束，即9个值都设定一个自定义值，且等式两边不相等，则right进行自适应调整。

    (2) 如果left和right都为auto，则首先left的值调整为包含块padding-left，然后考虑：

    • 如果width为auto，则绝对定位元素的宽度默认由内容撑开。
    • right的值计算方法：包含块可视区域大小 - 其他8个值。

    (3) 如果left = auto，right ≠ auto，则：

    • 如果width为auto，则绝对定位元素的宽度默认由内容撑开。
    • left的值计算方法：包含块可视区域大小 - 其他8个值。

    (4) 如果right = auto，left ≠ auto，则：

    • 如果width为auto，则绝对定位元素的宽度默认由内容撑开。
    • right的值计算方法：包含块可视区域大小 - 其他8个值。

    如果left/right至少有一个为auto，且将margin-left或margin-right设置为auto的话，则margin-left和margin-right的auto不起作用，会被置为零。

    (5) 如果left ≠ auto且right ≠ auto，则：

    • width、margin-left、margin-right至少一个为auto，类似于文档流中的水平方向布局等式：

    • 9个值都不为auto，如果发生过度约束，如（1）.

粘性定位 sticky

粘性定位目前主要拿来做吸顶效果，开启粘性定位元素的特点：

  1. 元素不会脱离文档流，元素的性质不会发生改变。
  2. 不论元素偏移到何处，原来占据的空间会被保留。
  3. 粘性定位的定位原点是第一个可滚动祖先的内容区左上角。
  4. 粘性定位的元素只会在包含块中显示⭐。

注意点：

  • 粘性定位元素的包含块的overflow属性值不可以是除visible以外的值，否则不会有粘性效果，只会等价于开启了相对定位relative。
  • 粘性定位元素必须指定top/bottom/left/right中的一个值，否则只会等价于开启了相对定位。

元素层级 z-index

前提：元素的position属性值不是static，即元素开启了定位。

可以通过设置z-index属性来指定元素的层级，z-index样式的值为一个整数（一般为非负整数），数值也越大表示优先在最上层显示。

注意：

  1. 若两个或多个不存在祖先-后代关系的元素的z-index值相同，则优先显示在HTML代码中靠后的元素。

  2. 绝大多数情况下，祖先元素永远不会盖住后代元素，即使后代元素z-index小于祖先元素的z-index。

    ❗特例：后代元素z-index设置为负数，而祖先元素没有设置z-index。

font-family

该样式用于设置元素的字体类型，是可继承样式⭐。

常用的默认值：

  1. serif 👈衬线字体
  2. sans-serif 👈 非衬线字体
  3. monospace 👈 等宽字体

font-family可以指定多个值，多个值之间用逗号隔开，优先生效靠前的，最后一个可选值一般是上述的默认值之一（因为被所有机器的上的浏览器所支持）。

字体颜色和大小

color：设置字体颜色，准确来说color是前景色。可继承样式⭐。

font-size：设置字体大小。可继承样式⭐。

行高 line-height

line-height是可继承样式 ⭐，对于块元素和内联元素设置line-height有不同的含义：

  1. 块元素设置行高，是指一行文本占用的实际高度，其样式会被后代元素继承。
  2. 内联元素设置行高，是指自身一行文本所占用的实际高度，但与其盒模型大小无关，如果其值大于自己包含块的行高，则会撑起自身所在行框的行高。

line-height的样式值可以为 20px、1em、1rem 等尺寸，也可以设置为一个数（代表是font-size的多少倍，即省略了单位em）。

⭐要点：

设置元素的font-size属性实际上设置的是字体框的高度，而非真是的字符高度（字符高度可能大于也可能小于字体框高度）。

行高line-height会在字体框的上下平均分配。

text-align

text-align属性一般设置在块元素上，其定义该块元素的行内内容（例如文字）如何相对于自身对齐。

text-align 并不控制块元素自己的对齐，只控制它的行内内容的对齐。

该样式是可继承样式⭐。

可选值：

  1. left 左对齐，默认值
  2. right 右对齐
  3. center 居中对齐
  4. justify 两端对齐

vertical-align

vertical-align是用于设置行内元素的行框与父元素行框对齐的方式。

行框是指元素line-height样式值形成的框。

vertical-align只能对行内元素、表格单元格元素生效，不能用它垂直对齐块级元素。

可选值：

  1. baseline 基线对齐，默认值

  2. top 子元素行框顶端与父元素行框顶端对齐

  3. middle 子元素文本基线与父元素文本中的“X”字符中心点高度对齐（一般不用，只在表格单元格中有使用）。

  4. bottom 子元素行框底端与父元素行框底端对齐

  • 条件2：子元素vertical-align等于top/bottom

    效果：子元素文本贴在父元素行框上边缘或下边缘。

用于解决一个bug：

前提：当使用div或其他块元素中包裹一个img标签，且div高度由img撑开

现象：由于img为替换元素，img底部为图片的基线，而vertical-align默认为baseline，这就导致img底部将和div的行框基线对齐，div被img撑开的实际高度大于img高度，出现一个外层边缘。

解决办法：

方法1，可以将img标签的vertical-align设置为top或bottom

放法2，可以将div的高度写死为img的高度。

white-space

white-space在CSS中的作用比较杂，不同white-space样式值会有不同的控制效果。

可选值：

  1. normal 👈默认值
  2. nowrap
  3. pre
  4. pre-wrap
  5. pre-line
可选值 对连续空白字符的处理策略 对回车符\r的处理策略 是否软换行（文本超出父元素宽度时是否自动换行）
nomal 压缩成一个空格 视为一个空白字符 是
nowrap 压缩成一个空格 视为一个空白字符 否
pre 保留所有空白字符 视为一个文本换行标志 否
pre-wrap 保留所有空白字符 视为一个文本换行标志 是
pre-line 压缩成一个空格 视为一个文本换行标志 是

根据需求选择合适的样式值：

  1. 如果需要一个段落的文本保持单行显示，则使用nowrap。
  2. 如果需要文本和HTML代码中一致，所见即所得，则使用pre。
  3. pre-line和pre-wrap需要根据具体情况进一步择取。

word-break

该样式值主要用于控制软换行时，对单词进行打断的策略。

可选值：

  1. normal ，👈 默认值，每个浏览器有自己默认的软换行策略，比如：
    • 在英文单词内不打断，如果一行末尾不够放下一个单词，则软换行显示。
    • 空格、连字符用于分割英文单词，可以在这两个字符后软换行。
    • 可以在任意的中文字符后面软换行。
  2. break-all，可以打断任意的单词，即可以在任何字符后面软换行。
  3. keep-all，中日韩文字等同于英文字符，作为单词的一部分，再按normal处理。

overflow-wrap/word-wrap

该样式值主要针对超长单词进行软换行控制。

overflow-wrap是CSS标准样式名，而word-wrap是其非标准别名，二者是同一样式，但尽量用标准的名字。

可选值：

  1. normal，超长单词会溢出父元素。
  2. break-word，会对超长单词进行折断换行。

① ② ③ ④ ⑤ ⑥

background-attachment

可选值：

  1. scroll 👈 默认值

    元素的背景图片会随着该元素在页面中的滚动而滚动，但是不会随着自身滚动条的滚动而滚动。

  2. local

    元素的背景图片会随着自身滚动条的滚动而滚动，但是不会随着该元素在页面中的滚动而滚动。

    另一个区别于scroll的点是：background-postion的百分比定位的参照尺寸发生了变化，background-attachment: local;时的百分比参照尺寸是元素background-origin规定的尺寸 + 可滚动尺寸。

  3. fixed

    元素的背景图片相对于页面布局视口进行定位，而与元素的位置无关。即background-position此时设定的偏移量是相对于视口而言的。

    但是该元素的背景图片只能在该元素范围内显示，如果背景图片最终的布局位置不被该元素所占有，则背景图片不显示在页面上。（部分或全部）

弹性盒子

引入弹性盒子的目的：提供一种更加有效的方式来对一个容器内的子元素进行排列、对齐和分配空白空间。

弹性盒子组成：弹性容器 + 弹性元素

display: flex/inline-flex;

  • 元素通过display: flex;成为块级弹性容器。
  • 元素通过display: inline-flex;成为行内弹性容器。
  • 弹性元素是指弹性容器的所有直接子元素。（既包括块元素，也包括行内元素，但不包括脱离文档流的子元素）
  • 一个元素既可以是弹性元素，也同时可以是弹性容器。弹性元素默认情况下会创建一个BFC。

弹性容器

flex-direction

作用：指定弹性容器中弹性元素的排列方式、伸缩方向，该方向为弹性盒子的主轴方向，另一个方向（垂直/水平）成为辅轴方向。

可选值：

  1. row 👈 默认值
  2. row-reverse 👈 S型软换行
  3. column
  4. column-reverse

flex-wrap

设置主轴方向剩余尺寸不够存放下一个弹性子元素时是否自动换行。

可选值：

  1. nowrap 👈 默认值，不换行
  2. wrap 换行
  3. wrap-reverse S型换行

如果指定换行，则flex-shrink将不再起作用。

flex-flow 简写属性

作用：flex-direction和flex-wrap的简写属性

默认值：flex-flow: row nowrap;

justify-content

作用：设置主轴方向上空白的分配方式。

可选值：

  1. flex-start，👈默认值，主轴方向上，元素尽可能有序靠前，空白在所有元素之后。
  2. flex-end，主轴方向上，元素尽可能按序靠后布局，空白在所有元素之前。
  3. center，主轴方向上，所有元素居中，空白平均分布到所有元素两侧。
  4. space-around，主轴方向上，空白平均分配到每个元素两侧，元素与元素之间距离是元素与父容器之间距离的两倍。
  5. space-between，主轴方向上，空白平均分配到元素与元素之间，元素与父容器之间无间隙。
  6. space-evently，主轴方向上，空白平均分配到元素与元素、元素与父容器之间，元素与元素之间距离等于元素与父容器之间距离。

align-content

作用：如果辅轴方向上有空白，该属性用于设置在允许弹性元素换行时，辅轴方向上空白的分配方式。

⭐该样式起作用的前提是：flex-wrap的属性值不为nowrap。

可选值：

假设flex-direction的值为: row，flex-wrap的值不是nowrap时，则一行弹性元素所占的高度称为行框。

行框的高度由一行弹性子元素中高度最高的弹性子元素为准。

  1. normal ，👈 默认值，辅轴方向上的空白平均分配到每个行框的下方。
  2. flex-start，辅轴方向上，所有行框尽量靠上，空白分配到所有行框的下方。
  3. flex-end，辅轴方向上，所有行框尽量靠下，空白分配到所有行框的上方。
  4. center，辅轴方向上，所有行框居中，空白平均分配到所有行框的上方和下方。
  5. space-around，辅轴方向上，空白平均分配到所有行框两侧，行框与行框之间的空白距离是行框与父容器之间距离的两倍。
  6. space-between，辅轴方向上，空白平均分配到所有行框之间，行框与父容器之间没有空白。
  7. space-evently，辅轴方向上，空白平均分配到所有行框以及行框和父容器之间，行框与行框之间的空白距离等于行框与父容器之间的距离。

align-items

作用：设置同一行框或同一列框上的弹性元素对齐方式。

可选值：

  1. stretch，👈 默认值，将辅轴方向上尺寸样式为auto的元素进行拉伸，以flex-direction: row;为例
    • 如果不配合flex-wrap: wrap;和align-content样式使用，则height: auto;的弹性子元素高度会拉满整个行框高度+空白区域。
    • 如果配合flex-wrap: wrap;以及align-content:normal;使用，则height: auto;的弹性子元素高度会拉满整个行框高度+空白区域。
    • 如果配合flex-wrap: wrap;以及除了normal以外的其他align-content的属性值使用时，则height: auto;的弹性子元素高度只会拉满整个行框高度。
  2. flex-start，同一行框内的弹性子元素上方对齐。
  3. flex-end，同一行框内的弹性子元素底端对齐。
  4. center，同一行框内的弹性子元素居中对齐。
  5. baseline，同一行框内的弹性子元素与父容器的本行基线对齐。

flex-grow

作用：为弹性盒子中所有的弹性子元素设置相同的flex-grow。

默认值：flex-grow: 0;

flex-shrink

作用：为弹性盒子中所有的弹性子元素设置相同的flex-shrink。

默认值：flex-shrink: 1;

注意，当弹性容器的flex-wrap样式值不为nowrap时，所有的flex-shrink将不起作用。

flex-basis

作用：为弹性盒子中所有的弹性子元素设置相同的flex-basis。

默认值：flex-basis: auto;

flex 简写属性

作用：为弹性盒子中的所有子元素设置相同的flex-grow、flex-shrink和flex-basis。

默认值：flex: 0 1 auto;

弹性元素

flex-grow

作用：弹性元素在主轴方向上的增长系数。弹性元素所在行框所有弹性元素的原始尺寸之和小于父容器内容区尺寸时，拉伸弹性元素。

默认值：flex-grow: 0;

需要注意的是，即使一个弹性元素在主轴方向上的原始尺寸等于零，在设置flex-grow大于0时，它也会分配到空白。

flex-shrink

作用：弹性元素在主轴方向上的压缩系数。弹性元素所在行框所有弹性元素的原始尺寸之和大于父容器内容区尺寸时，压缩弹性元素。

默认值：flex-shrink: 0;

每个弹性元素主轴方向压缩量的计算方法：

需要注意的是，即使一个弹性元素在主轴方向上的原始尺寸等于零，那么它就不会被压缩。

flex-basis

作用：设置元素在主轴上的原始尺寸。

默认值：flex-basis: auto;

和元素width的关系

  1. 弹性元素的flex-basis为auto时，弹性元素的原始宽度由width决定。
  2. 弹性元素的flex-basis为大于0的值时，弹性元素的原始宽度由flex-basis决定，无视width。
  3. 弹性元素的flex-basis为0时，弹性元素的原始宽度默认由内容撑开，无视width。

flex 简写属性

可以使用一个，两个或三个值来指定 flex属性。

单值语法: 值必须为以下其中之一

  • 一个无单位数：它会被当作flex:<number> 1 0; <flex-shrink>的值被假定为1，然后<flex-basis> 的值被假定为0。❗
  • 一个有效的宽度width值：它会被当作 <flex-basis>的值。
  • 关键字none，auto或initial.

双值语法: 第一个值必须为一个无单位数，并且它会被当作 <flex-grow> 的值。第二个值必须为以下之一：

  • 一个无单位数：它会被当作 <flex-shrink> 的值。
  • 一个有效的宽度值: 它会被当作 <flex-basis> 的值。

三值语法:

  • 第一个值必须为一个无单位数，并且它会被当作 <flex-grow> 的值。
  • 第二个值必须为一个无单位数，并且它会被当作 <flex-shrink> 的值。
  • 第三个值必须为一个有效的宽度值， 并且它会被当作 <flex-basis> 的值。

建议：三个都写。⭐
